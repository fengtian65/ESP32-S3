# ESP32-S3 对接Coze智能体温湿度建议项目
本项目基于**ESP32-S3**开发板，实现**DHT11温湿度采集** + **按键触发** + **WiFi联网** + **Coze（扣子）智能体流式API调用**功能，按下按键后自动采集温湿度数据，向Coze智能体发送请求并获取个性化生活建议，全程通过串口打印日志及智能体回复，是物联网设备对接大模型智能体的入门实战项目。

## 项目特性
1. WiFi STA模式自动连接，断连后自动重连，提供连接状态查询接口；
2. DHT11温湿度采集：带**互斥锁**防数据竞争、**3次重试机制**、**校验和验证**，采集稳定；
3. 按键触发：GPIO外部中断（下降沿）+ **事件队列** + **50ms消抖**，避免误触；
4. Coze API对接：支持**SSE流式数据解析**、前置网络连通性检测、JSON请求体动态构建、流式回复逐字拼接；
5. 完善的错误处理：对WiFi未连接、DHT11读取失败、API请求超时/非200响应等场景做异常处理；
6. 跨端测试支持：提供Python脚本`test.py`，可在电脑端快速验证Coze API的可用性，排除平台/网络问题。

## 硬件接线
所有接线均采用**3.3V供电**（避免5V烧毁ESP32-S3引脚），核心接线如下，引脚可在对应头文件中修改：
| 模块引脚 | ESP32-S3引脚 | 备注                     |
|----------|--------------|--------------------------|
| DHT11-VCC | 3.3V         | 严禁接5V                 |
| DHT11-GND | GND          | 共地                     |
| DHT11-DATA | GPIO5      | 可在`dht11.h`中修改`DHT11_GPIO` |
| 按键一端  | GPIO11       | 可在`button.c`中修改`BUTTON_GPIO` |
| 按键另一端  | GND          | 代码中已开启GPIO上拉，按下为低电平 |

## 软件环境
1. **ESP-IDF**：推荐v5.0及以上版本（项目基于ESP-IDF原生API开发，无第三方库依赖）；
2. **Python3**：用于运行`test.py`，验证Coze API的可用性，需安装依赖`pip install requests json`；
3. **Coze平台**：已完成智能体创建、**沙箱部署**，并获取到**流式API地址**、**Token**、**Project ID**；
4. 串口工具：ESP-IDF Monitor/串口助手（波特率115200，8N1）。

## 快速开始
### 步骤1：Coze平台配置（关键）
确保Coze智能体已**沙箱部署成功**，并将平台获取的以下信息更新到代码中（`coze_api.c`）：
```c
#define COZE_URL       "你的Coze流式API地址"  // 如https://xxx.coze.site/stream_run
#define COZE_SESSION   "自定义会话ID"        // 如ESP32_S3_CUSTOM_SESSION_001
#define COZE_PROJECT   "你的Project ID"      // 数字字符串
static const char *COZE_TOKEN = "你的Coze Token";  // 平台生成的Bearer Token
```
> 验证：修改`test.py`中的同名配置，运行`python test.py`，若能正常获取智能体回复，说明Coze配置无误。

### 步骤2：WiFi配置
修改`wifi_conn.h`中的WiFi名称和密码，适配你的本地WiFi（仅支持2.4G WiFi，ESP32-S3默认不支持5G）：
```c
#define WIFI_SSID      "你的WiFi名称"  // 2.4G SSID
#define WIFI_PASS      "你的WiFi密码"  // WiFi密码
```

### 步骤3：代码编译与烧录
1. 打开ESP-IDF终端，进入项目根目录；
2. 设置开发板目标：`idf.py set-target esp32s3`；
3. 编译项目：`idf.py build`；
4. 烧录并打开串口监视器：`idf.py flash monitor`（烧录前需将ESP32-S3进入下载模式）。

### 步骤4：功能测试
1. 串口监视器中查看WiFi连接状态，显示`Connected! IP: xxx.xxx.xxx.xxx`即为联网成功；
2. 按下连接在GPIO11的按键，开发板会自动执行：**DHT11采集温湿度** → **网络连通性检测** → **调用Coze API**；
3. 串口会依次打印采集的温湿度、API请求日志，最终输出Coze智能体的**完整生活建议**。

## 核心功能说明
### 1. WiFi连接与重连
- 基于ESP-IDF原生`esp_wifi` API实现STA模式连接；
- 注册WiFi/IP事件回调，断连后1秒自动重连，通过`wifi_is_connected()`提供全局连接状态查询；
- 关闭WiFi节能模式，保证网络通信稳定性。

### 2. DHT11温湿度采集
- 单总线通信协议，自定义微秒级延时（关闭编译器优化，保证延时精度）；
- 加**互斥锁**（`Semaphore`），防止多任务同时读取DHT11导致数据错乱；
- 采集后做**校验和验证**，校验失败则自动重试（最多3次）；
- 采集失败后恢复GPIO引脚状态，避免总线异常。

### 3. 按键触发机制
- 采用**GPIO外部中断（下降沿）** 触发，避免轮询占用CPU资源；
- 中断服务函数中通过**FreeRTOS事件队列**传递按键事件，实现**中断与任务解耦**（中断中不做复杂操作）；
- 主任务中对按键事件做**50ms消抖**，排除机械按键的误触干扰。

### 4. Coze智能体API调用
- **前置检查**：调用API前先检查Token是否为空、WiFi是否连接，避免无效请求；
- **网络检测**：自动检测Coze域名的DNS解析和TCP 443端口连通性，提前排查网络问题；
- **SSE流式解析**：按Coze SSE协议规范，解析`data:`开头的流式数据，逐字拼接智能体回复；
- **超时处理**：SSE数据读取设置20秒超时，避免无限阻塞；
- **内存管理**：动态扩容存储智能体回复，请求结束后释放所有申请的内存，无内存泄漏。

## 项目代码结构
项目采用**模块化编程**，按功能拆分文件，低耦合、高可维护，核心文件及作用如下：
```
├── main.c          # 主任务入口，整合所有模块，处理按键事件并触发API调用
├── wifi_conn.c/h   # WiFi驱动模块，实现WiFi连接、重连、状态查询
├── dht11.c/h       # DHT11驱动模块，实现温湿度采集、校验、重试
├── button.c/h      # 按键驱动模块，实现中断初始化、事件队列、消抖
├── coze_api.c/h    # Coze API交互模块，实现SSE请求、流式解析、回复拼接
└── test.py         # Python端Coze API测试脚本，快速验证平台/网络配置
```
### 核心文件职责
| 文件          | 核心职责                     | 对外接口                     |
|---------------|------------------------------|------------------------------|
| wifi_conn.c/h | WiFi连接与重连               | `wifi_conn_init()`、`wifi_is_connected()` |
| dht11.c/h     | DHT11温湿度采集              | `dht11_init()`、`dht11_read()` |
| button.c/h    | 按键中断与事件处理           | `button_init()`、`button_get_event_queue()` |
| coze_api.c/h  | Coze API调用与流式解析       | `coze_send_request(float temp, float humid)` |
| main.c        | 主任务调度                   | 无对外接口，项目入口         |

## 注意事项
1. **Coze平台相关**
   - 确保智能体处于**沙箱部署在线**状态，离线会导致API请求失败；
   - Coze Token有有效期，若请求报认证错误，需重新生成Token；
   - 仅支持**流式API（stream_run）**，非流式API需修改`coze_api.c`中的请求逻辑。
2. **硬件相关**
   - DHT11必须接**3.3V**，ESP32-S3的GPIO为3.3V电平，接5V会烧毁引脚；
   - 按键接线为**上拉输入接GND**，若改为下拉输入接VCC，需修改`button.c`中的`intr_type`为`GPIO_INTR_POSEDGE`；
   - 杜邦线连接需牢固，接触不良会导致DHT11读取失败、按键无响应。
3. **软件相关**
   - ESP-IDF编译前需确认目标芯片为`esp32s3`，否则会出现烧录/运行异常；
   - 项目无第三方库依赖，直接用ESP-IDF原生API编译即可，无需额外安装组件；
   - 串口波特率为**115200**，若串口乱码，需调整串口工具的波特率。

## 常见问题排查
### 问题1：WiFi连接失败，提示`Disconnected, reconnecting...`
- 排查1：确认WiFi为**2.4G**，ESP32-S3默认不支持5G WiFi；
- 排查2：检查`wifi_conn.h`中的SSID和密码是否输入正确（区分大小写）；
- 排查3：WiFi信号过弱，将开发板靠近路由器；
- 排查4：路由器开启了MAC地址过滤，将ESP32-S3的MAC地址加入白名单。

### 问题2：按下按键，DHT11读取失败，提示`Checksum failed`/`Read timeout`
- 排查1：DHT11接线是否正确，DATA引脚是否为GPIO5（或自定义的引脚）；
- 排查2：DHT11供电是否为3.3V，供电不足会导致数据采集异常；
- 排查3：杜邦线接触不良，重新插拔DHT11的DATA引脚；
- 排查4：环境温度/湿度是否在DHT11量程内（0~50℃，20%~90%RH）。

### 问题3：WiFi已连接，Coze API调用失败，报TCP连接失败
- 排查1：运行`test.py`，若电脑端也无法请求，说明Coze平台/配置问题（Token/Project ID/URL错误）；
- 排查2：ESP32是否能访问外网，可修改`coze_api.c`中的测试地址，检测`www.baidu.com:80`连通性；
- 排查3：路由器开启了防火墙/代理，关闭后重试；
- 排查4：Coze API地址的域名解析失败，可尝试直接替换为IP地址（从`test.py`的日志中获取）。

### 问题4：Coze API返回200，但无智能体回复，提示SSE读取超时
- 排查1：确认Coze智能体的**prompt提示词**是否正确，是否能对温湿度数据做出回复；
- 排查2：ESP32的网络带宽不足，关闭其他占用网络的设备，重新测试；
- 排查3：修改`coze_api.c`中的SSE超时时间（默认20秒），适当增大；
- 排查4：检查Coze智能体的响应速度，若电脑端`test.py`回复也慢，需优化智能体逻辑。

### 问题5：curl测试Coze API报503错误，但Python脚本正常
- 核心原因：curl命令**缺少流式请求头**/**无合法JSON请求体**/**Token为占位符**，并非平台服务问题；
- 解决方案：使用项目中**修正后的curl命令**（带`Accept: text/event-stream`等头+完整JSON请求体），替换真实Token后测试。
